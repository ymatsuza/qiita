金額は **¥3,120**。

……¥3,120か。

**着想からアプリ完成まで2時間。びっくりドンキーで食事しながらの2時間でした。**

## Claude Cowork（Dispatch）とは

Claude Desktop の **Cowork タブ**（旧称: Dispatch）は、スマホから自宅や職場のMacをリモート操作して Claude Code を走らせられる機能です。

```
スマホ（外出先）
    ↓ Claude Cowork で接続
Mac（自宅）
    ├── Claude Code が動く
    ├── MCP 経由で各種サービスとも連携
    └── コード生成・ビルド・Git push まで全部やってくれる
```

自分の構成はこんな感じです：

| 項目 | 内容 |
|------|------|
| スマホ | Android Pixel |
| Mac | Mac mini（自宅常時起動） |
| プラン | Claude Team |
| 繋いでいるMCP | Slack / Notion / GitHub / Filesystem / Gmail / Calendar / Asana / Figma など |

### Mac のスリープ設定だけ忘れずに

Cowork の設定パネルにある **「スリープしない」トグルをオンにするだけ**。

<img src="https://raw.githubusercontent.com/ymatsuza1128/qiita/main/images/cowork-ui.png" width="500">

これさえやれば、あとはスマホの Claude アプリから Cowork タブを開くだけで繋がります。

<details><summary>2. JGit ラッパ：commit & push</summary>

GitHub への push は Personal Access Token を `UsernamePasswordCredentialsProvider` の **ユーザー名側** に渡すのが正解（パスワードは空）。地味にハマりがちなところを Claude が当たり前のようにやっていました。

```kotlin
suspend fun commitAndPush(
    message: String,
    author: PersonIdent,
    branch: String,
): CommitInfo? = withContext(Dispatchers.IO) {
    Git.open(workTreeProvider()).use { git ->
        git.add().addFilepattern(".").call()
        // Also stage deletions, which `add .` does not catch in JGit.
        git.add().addFilepattern(".").setUpdate(true).call()
        val status = git.status().call()
        if (status.isClean) return@withContext null
        val commit = git.commit()
            .setMessage(message)
            .setAuthor(author)
            .setCommitter(author)
            .call()
        val pushCmd = git.push()
            .setRemote("origin")
            .add(branch.ifBlank { "main" })
        tokenProvider()?.let { pushCmd.setCredentialsProvider(credentials(it)) }
        pushCmd.call()
        CommitInfo(
            sha = commit.name,
            shortMessage = commit.shortMessage,
            authorName = commit.authorIdent.name,
            authorEmail = commit.authorIdent.emailAddress,
            whenEpochSec = commit.commitTime.toLong(),
        )
    }
}
private fun credentials(token: String) =
    UsernamePasswordCredentialsProvider(token, "")
```

## ハマりポイント：外出中は「許可ダイアログ」に答えられない

Cowork を外出先から使って気づいた一番の落とし穴がこれです。

Mac 側で `sudo` が必要なコマンドや、GUI の許可ダイアログが出る操作は、スマホ側から対応できません。

対処法として Claude が自分で考えてくれたのが：

```
sudo が必要なルートを回避する
（例: brew install --cask ではなく brew install openjdk）
→ どうしても詰まりそうなら NOTES.md にハンドオフを書いて停止
```

「外出中なので許可ダイアログには答えられない」と一度伝えておくと、その制約を保持したまま進めてくれます。
